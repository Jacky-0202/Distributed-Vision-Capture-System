### 1. 邏輯位址 (1-based) vs 協議位址 (0-based)

Modbus 的位址存在兩種表述方式：

- **邏輯位址（文件常用）：** 為了方便人類閱讀，手冊常從 `1` 開始編號（例如第 1 個暫存器）。著名的 **Modicon 地址** 會加上前綴區分性質，如 `40001` 代表第一個 Holding Register。
- **協議位址（程式常用）：** 在實際的封包傳輸中，位址是從 `0` 開始的二進位數值 。
- **轉換規則：** 程式中的位址 = 手冊位址 - 1。例如要讀取 `40001`，Python 程式碼中應填寫 `address=0`。

### 2. 台達 PLC 的特定定址映射 (Delta DVP Series)

根據您的手冊，台達 PLC 將其內部元件映射到了特定的 Modbus 十六進制位址 。這是您在寫 Python 時最重要的參考表：

|**PLC 元件**|**範圍 (十進制編號)**|**Modbus 協議位址 (Hex)**|**功能碼 (FC)**|
|---|---|---|---|
|**D 暫存器**|D0 ~ D999|**`1000H`** ~ `13E7H`|03, 06, 16|
|**M 繼電器**|M0 ~ M511|**`0800H`** ~ `09FFH`|01, 05, 15|
|**Y 輸出點**|Y0 ~ Y27|**`0500H`** ~ `051BH`|01, 05, 15|
|**X 輸入點**|X0 ~ X27|**`0400H`** ~ `041BH`|02|

> **計算範例：**
> 
> 如果您想讀取 **D100**：
> 
> 1. D0 的起始位址是 `1000H` (4096)。
> 2. D100 的偏移量是 100。 
> 3. Python `pymodbus` 位址 = $4096 + 100 = 4196$ (或 `0x1064`) 。

### 3. 八進制轉換邏輯 (X 與 Y 點)

台達 PLC 的輸入 (X) 與輸出 (Y) 是採用 **八進制 (Octal)** 編號 。但在 Modbus 通訊中，它們會被轉換成連續的 **十六進制** 地址：

- **PLC 編號：** X0, X1... X7, **X10**, X11... (沒有 X8, X9) 。
- **轉換公式：** $Modbus 位址 = 起始位址 + (編號的十進制值)$。
- **案例：** 讀取 **X10**：
    
    - 八進制 `10` 轉換為十進制是 `8`。
    - X0 起始位址是 `0400H`。
    - Modbus 實際位址 = `0400H + 8 = 0408H`。
