### 摘要

這三個指令負責在本地儲存庫 (Local Repo) 與遠端儲存庫 (Remote Repo) 之間搬運「快照紀錄」。

---

### 指令定義與差異

- **git push (推)**：
    
    - **動作**：將本地的新紀錄上傳到遠端。
    - **目的**：分享你的進度或備份到雲端。
    - **指令**：

第一次上傳
```bash
git push -u origin main
```
通用
```bash
git push
```
        
- **git fetch (抓)**：
    
    - **動作**：從遠端下載最新的紀錄到本地，但**不會**自動合併到你正在寫的檔案中。
    - **目的**：「看看」別人在雲端改了什麼，而不影響自己目前的工作。
        
- **git pull (拉)**：
    
    - **動作**：`fetch` + `merge` 的組合技。先下載紀錄，然後立即與本地分支合併。
    - **目的**：快速同步雲端進度。

---
### 操作邏輯 (SOP)

1. **上傳前先更新**：在 `push` 之前，養成先 `pull` 的習慣，確保你是在最新的基礎上開發，避免衝突。
2. **謹慎開發用 fetch**：如果你不確定雲端的變動是否會弄亂你的程式碼，先用 `fetch` 觀察，確認沒問題再手動 `merge`。
3. **推送到遠端**：確認本地紀錄都 commit 好了，執行 `push` 同步。

> 如果是私人的 repo 每次輸入金鑰很麻煩可以設定成 global
> `git config --global credential.helper store`
