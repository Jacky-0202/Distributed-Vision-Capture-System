### 摘要

將程式碼的變動從「編輯現場」轉化為「正式快照」的過程。這不僅是存檔，更是專案開發邏輯的梳理。

---
### 精準標記 (git add)

- **核心功能**：將工作區的變更移動到暫存區（Staging Area）。
- **常用方式**：
    
    - `git add <file>`：加入特定檔案。
    - `git add .`：將當前目錄下所有變更加入暫存。
        
    - `git add -p`：**進階推薦**。互動式選擇「片段 (Hunk)」，可以只提交檔案中的某幾行程式碼，實現極致的原子化。

---
### 正式提交 (git commit)

- **核心功能**：將暫存區的快照永久存入儲存庫。

最常用的提交方式：
```bash
git commit -m "message"
```

 在編輯器中顯示變更細節
```bash
git commit -v
```

- **提交訊息 (Commit Message) 藝術**：
    
    - **原則**：好的訊息應描述「為什麼改」，而非「改了什麼」。
    - **格式建議**：`feat: 增加登入功能` 或 `fix: 修復首頁跑版`。

---
### 操作邏輯

1. **挑選**：透過 `git add` 決定這一次「存檔箱子」裡要裝什麼。
2. **檢閱**：使用 `git status` 確認箱子裡的清單是否正確。
3. **封箱**：使用 `git commit` 貼上標籤並存入倉庫。
