## 摘要

將不同分支的變動整合在一起的過程。如果兩邊改到同一個檔案的同一行，Git 會停下來要求你手動判斷要留誰，這就是「衝突」。

---
### 合併操作 (git merge)

- **核心功能**：將指定分支的內容併入「當前所在」的分支。
- **操作邏輯**：
    
    1. 先切換回接收者：`git switch main`
    2. 執行合併：`git merge test_branch`
        
- **快進式合併 (Fast-forward)**：如果 `main` 在這期間沒動過，Git 會直接把 `main` 的標籤移到最新位置，不產生額外節點。

---
### 衝突處理 (Conflict Resolution)

- **發生原因**：兩個分支修改了同一個檔案的**同一行**。
- **Git 的反應**：合併會暫停，檔案內容會被標註衝突區塊：
```plaintext
<<<<<<< HEAD (目前分支的內容)
我的正式版本代碼
=======
分支上的實驗性代碼
>>>>>>> test_branch (來源分支的內容)
```

**解法**：

1. 手動刪除 `<<<`, `===`, `>>>` 標籤，並保留正確的代碼。
2. 執行 `git add <檔案>` 標示衝突已解決。
3. 執行 `git commit` 完成合併流程。
