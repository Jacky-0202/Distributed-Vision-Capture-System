## 摘要

掌握專案當下的檔案狀態，並回溯已發生的提交歷史，是進行任何進階操作（如回退、合併）的前提。

---
## 狀態檢視 (git status)

- **核心功能**：顯示工作區與暫存區的差異。
- **顯示資訊**：
    
    - 哪些檔案被修改了但還沒 `add`。
    - 哪些檔案已經 `add` 進入暫存區準備 `commit`。
    - 目前在哪個分支上。
        
- **常用參數**：
    
    - `git status -s`：以簡短模式顯示（精簡標記如 `M` 代表修改、`??` 代表未追蹤）。

---
## 紀錄回溯 (git log)

- **核心功能**：列出儲存庫中所有的提交紀錄。
- **顯示資訊**：
    
    - 提交的雜湊值 (Commit ID / SHA-1)。
    - 作者與日期。
    - 提交訊息 (Commit Message)。
    
- **常用參數**：
    
    - `git log --oneline`：每一條紀錄縮成一行，適合快速瀏覽。
    - `git log --graph`：以圖形化顯示分支的合併狀態。
    - `git log -p <file>`：查看特定檔案在過去的詳細修改內容。

---
## 操作邏輯

1. **動手前**：執行 `git status` 確認目前的狀態是否「乾淨」。
2. **提交後**：執行 `git log --oneline` 確認剛才的操作是否成功存入歷史。
