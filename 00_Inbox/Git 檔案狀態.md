## 摘要

Git 根據檔案在「三階段模型」中位置的不同，賦予檔案的四種邏輯標籤。

---
## 四大狀態類型

- **未追蹤 (Untracked)**：新建立的檔案，Git 還不知道它的存在。
- **已修改 (Modified)**：檔案已被更動，但尚未加入暫存區。
- **已暫存 (Staged)**：檔案已執行 `git add`，準備進入下一個快照。
- **未修改 (Unmodified)**：已提交至儲存庫，且內容與工作區完全一致的檔案。

---
## 狀態轉換流程

1. **新增檔案**：狀態為 `Untracked`。
2. **執行 git add**：由 `Untracked` 或 `Modified` 轉為 `Staged`。
3. **執行 git commit**：由 `Staged` 轉為 `Unmodified`（代表已正式入庫）。
4. **再次修改檔案**：由 `Unmodified` 轉回 `Modified`。

---
## 核心價值

- **明確身份**：
	讓你透過 `git status` 瞬間掌握哪些檔案需要被打包，
	哪些檔案只是暫時生成的雜物。

- **安全防護**：Git 只會記錄處於 `Staged` 狀態的檔案，未被追蹤的檔案不會被意外提交。
