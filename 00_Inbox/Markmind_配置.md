## 摘要

利用 Markmind 的「全功能心智圖」特性，實現 Git 知識體系的雙向編輯（大綱 ↔ 心智圖），並確保與兩層扁平化系統高度相容。

---
### 🔐 核心觸發設定 (The YAML)

Markmind 依賴特定的 YAML 屬性來切換模式。請將此區塊置於筆記最上方：

- **右鍵選單（最推薦）**：在筆記任何地方點擊右鍵，選擇 `More options` -> `Open as mindmap`。
- **指令面板**：按下 `Ctrl/Cmd + P`，輸入 `Markmind: Open as mindmap`。
- **狀態標籤**：如果 YAML 設為 `markmind-type: markdown`，這代表這份文件本質是 Markdown，但你可以隨時「手動切換」成心智圖視角。

```yaml
---
markmind-type: markdown
---
```

---
## ⚙️ 全域與局部設定 (Settings)

進入 `Settings` -> `Markmind` 或在心智圖右鍵選單中調整：

- **預設編輯模式 (Default Mode)**：`Mindmap`（若希望打開即是圖表）。
    
- **節點連結風格 (Line Style)**：
    
    - 推薦 `Orthogonal` (直角) 或 `Rounded` (圓角)。
    - _工程師推薦選 **直角**，在處理 Git 複雜分支邏輯時最整齊。_
        
- **佈局方向 (Layout)**：
    - `Logical Structure (Right)`：向右增長，適合呈現「操作流程」。
    - `Mind Map`：放射狀，適合呈現「概念發散」。

## 效率快捷鍵與操作 (Hotkey

| **類別**   | **動作**  | **快捷鍵**           | **說明**             |
| -------- | ------- | ----------------- | ------------------ |
| **節點增減** | 新增子節點   | `Tab`             | 在目前節點下方建立次級知識點     |
|          | 新增同級節點  | `Enter`           | 建立一個水平並列的邏輯區塊      |
|          | 刪除節點    | `Delete`          | 移除選中的節點及其子項目       |
| **內容編輯** | 編輯文字    | `Space`           | 快速進入文字編輯模式         |
|          | 完成編輯    | `Enter`           | 編輯完文字後儲存並退出編輯狀態    |
|          | 節點內換行   | `Shift` + `Enter` | 在同一個節點內進行多行輸入      |
| **視覺整理** | 折疊 / 展開 | `/`               | 收納子節點，保持地圖畫面清爽     |
|          | 復原操作    | `Ctrl` + `Z`      | 撤銷上一步的移動或刪除        |
| **模式切換** | 視圖切換    | `Alt` + `M`       | **核心鍵**：在筆記與心智圖間跳轉 |

## 🔗 連結與附件規範 (Link Protocol)

為了維持地圖整潔並連結你的原子筆記：

1. **內部連結**：直接輸入 `[[筆記名稱]]`。
    
    - _Markmind 會自動將其轉化為可點擊的連結，點擊即可開啟原子筆記。_
        
2. **圖片嵌入**：使用 `![[${notename}_${date}.png]]`。
    
    - _圖文並茂是 Markmind 的強項，圖片會直接浮現在節點旁。_

---
## 🚀 雙視窗工作流 (Dual-Window)

這是在 Ubuntu 環境下最強大的使用方式：

1. **左側 (Source)**：編輯 Markdown 原始碼。負責精確調整路徑 `[ ]( )` 與文字描述。
2. **右側 (View)**：開啟 `Open as mindmap`。負責全局導航。
