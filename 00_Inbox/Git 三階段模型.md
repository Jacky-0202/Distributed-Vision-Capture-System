## 摘要

Git 檔案在被正式記錄前，依序流轉的三個邏輯區域。

---
## 三個區域

- **工作區 (Working Directory)**：
	你電腦上的實體資料夾。你在這裡新增、修改、刪除檔案。這代表「檔案目前的樣子」。
    
- **暫存區 (Staging Area / Index)**：
	一個預備提交的清單。它記錄了你「打算」在下一次存檔中包含哪些變動。
    
- **儲存庫 (Repository)**：
	Git 用來存放專案中繼資料與物件資料庫的地方（即 `.git` 資料夾），存放著所有已確認的歷史快照。

---
## 操作邏輯

1. **修改**：在工作區編輯檔案。
2. **標記 (git add)**：將檔案從工作區加入暫存區，表示「這部分我等等要存」。
3. **提交 (git commit)**：將暫存區的檔案打包，永久存入儲存庫。
