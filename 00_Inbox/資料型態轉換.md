## 摘要

在 Modbus 通訊中，**資料型態轉換（Data Type Conversion）** 是軟最核心的工作。
因為 Modbus 協議的最小單位是 **16 位元暫存器（Register）**，而 Python 的資料型態（如 Float, 32-bit Int）通常比這大，且存儲方式不同。

---
## 1. 核心概念：16-bit 的限制與組合

- **單個暫存器 (16-bit)：** 只能表示 $0 \sim 65535$ (無正負號) 或 $-32768 \sim 32767$ (有正負號)。
- **組合暫存器 (32-bit)：** 當你需要表示浮點數（Float）或較大的整數（DInt）時，必須佔用 **2 個連續的暫存器**。

---
## 2. 字節順序與字詞交換 (Endianness & Word Swap)

這是開發時最常遇到的「亂碼」來源。當 32 位元資料拆成 4 個 Byte 傳輸時，順序可能完全不同：

- **Big-Endian (大端模式):** 高位字節在前（ABCD）。
- **Little-Endian (小端模式):** 低位字節在前（DCBA）。
- **Word Swap (字詞交換):** 在 32 位元通訊中，有時 Byte 順序對了，但兩個 16-bit Register 的順序反了（CDAB）。

---
## 3. Python 實戰

若要僅使用 `import serial` 實作讀取台達 PLC D100 開始的 32 位元浮點數，你需要親自處理 **Modbus 封包組裝**、**CRC 校驗** 以及最關鍵的 **位元組順序對調（Word Swap）**。
台達 PLC 的 32 位元浮點數通常遵循：**D100 存放低位字詞 (Low Word)**，**D101 存放高位字詞 (High Word)**。

#### 範例：讀取 32 位元浮點數 (Float)

假設你要從台達 PLC 的 D100 讀取一個 32 位的浮點數（佔用 D100, D101）：

```python
import serial
import struct

# CRC-16 Modbus 計算函數
def calculate_crc(data):
    crc = 0xFFFF
    for pos in data:
        crc ^= pos
        for i in range(8):
            if (crc & 1) != 0:
                crc >>= 1
                crc ^= 0xA001
            else:
                crc >>= 1
    return crc.to_bytes(2, 'little')

# 1. 初始化序列埠 (請根據實際 COM Port 修改)
ser = serial.Serial(
    port='COM3', 
    baudrate=9600, 
    bytesize=8, 
    parity='E',  # 台達預設通常是 7, E, 1 (ASCII) 或 8, E, 1 (RTU)
    stopbits=1, 
    timeout=1
)

# 2. 組裝 Modbus RTU 請求
# 站號: 01 | 功能碼: 03 | 地址: 10 64 | 數量: 00 02
slave_id = 0x01
func_code = 0x03
address = 0x1064
count = 0x0002

request = struct.pack('>BBHH', slave_id, func_code, address, count)
request += calculate_crc(request)

# 3. 發送請求並讀取回傳
# 預期回傳長度: 1(ID) + 1(FC) + 1(長度) + 4(資料) + 2(CRC) = 9 Bytes
ser.write(request)
response = ser.read(9)

if len(response) == 9:
    # 4. 解析資料與轉換浮點數
    # response[3:5] 是 D100 (低位字詞), response[5:7] 是 D101 (高位字詞)
    # IEEE 754 標準浮點數解析需要 [高位字詞][低位字詞] 的順序
    
    d100_bytes = response[3:5]
    d101_bytes = response[5:7]
    
    # 重新拼接: 高位在前的 4 Bytes (ABCD 順序)
    raw_float_bytes = d101_bytes + d100_bytes
    
    # 使用 struct.unpack 轉為 Python 浮點數
    # >f 代表 Big-Endian Float
    float_value = struct.unpack('>f', raw_float_bytes)[0]
    
    print(f"D100-D101 讀取到的浮點數為: {float_value:.2f}")
else:
    print("通訊逾時或封包長度錯誤")

ser.close()
```
