### ⚠️ 核心前提：連線成功的三大鐵律

在執行任何 SSH 指令前，請先檢查這三點，否則只會看到 `Connection timeout`。

1. **同一個網域 (Same Subnet)**：
    
    - Ubuntu 主機與 CM4 必須連接到同一個路由器或熱點。
    - **檢查方式**：Ubuntu 的 IP 是 `192.168.34.x`，CM4 也必須是 `192.168.34.y`。
        
2. **隱藏 SSID 的啟動陷阱**：
    
    - **問題**：隱藏 SSID 不會主動廣播，CM4 開機後如果沒能成功「主動握手」，它就不會出現在區網內，導致 SSH 找不到目標。
    - **對策**：確保 CM4 已經有「連線成功紀錄」。如果連不上，可能需要接螢幕先手動連一次，或確認 `network-config` 中的 `scan_ssid=1` 有確實運作。
        
3. **IP 位址的時效性**：
    
    - WiFi 分配的 IP 可能會變動。連線失敗時，第一步永遠是先 `ping` 看看對象還在不在。

### 🚀 SSH 快速操作指南

#### 1. 基礎登入

```
ssh pi@192.168.34.144
```

#### 2. 免密碼金鑰設定（推薦）

在 Ubuntu 終端機執行：

```
ssh-keygen -t ed25519       # 產生金鑰 (一路 Enter)
ssh-copy-id pi@192.168.34.144  # 把鑰匙交給 CM4
```

#### 3. 快速撥號設定 (`~/.ssh/config`)

編輯設定檔，以後打 `ssh cm4` 就好：

```
Host cm4
    HostName 192.168.34.144
    User pi
```
