### 1. 通訊參數 (Physical Layer)

這部分確保 PLC 的 COM Port 能跟 CM4 對接成功。

|**項目**|**設定值**|**備註**|
|---|---|---|
|**傳輸介面**|RS-485 (Half Duplex)|連接 CM4 的 `/dev/ttyUSB0`|
|**站號 (Slave ID)**|**1** 或 **8**|CM4 目前採「雙站號」監聽以配合 HMI 預設值|
|**鮑率 (Baudrate)**|**19200**||
|**通訊格式**|**8, E, 1**|8 Data bits, Even Parity, 1 Stop bit|

### 2. 暫存器定義 (Register Map)

PLC 使用 `MODRW` 指令存取的位址定義。

| **Modbus 位址** | **PLC 位址** | **名稱**       | **讀/寫** | **數值定義與說明**                                       |
| ------------- | ---------- | ------------ | ------- | ------------------------------------------------- |
| **0000H**     | **40001**  | **複合拍照指令**   | 寫入 (W)  | **低位元(Bit 0-3)**: 光源模式<br>**高位元(Bit 4-11)**: 相機遮罩 |
| **0002H**     | **40003**  | **系統狀態**     | 讀取 (R)  | **1**: Running (處理中)<br>**2**: OK / Ready (閒置/完成) |
| **0009H**     | **40010**  | **Watchdog** | 寫入 (W)  | 心跳包，PLC 固定寫入 **1** 確保連線                           |
| **0010H**     | **40017**  | **版本號**      | 讀取 (R)  | 固定回傳 **100** (代表 v1.0.0)                          |

### 3. 指令數值拆解 (Command Bits)

針對 **40001** 的數值計算公式：`Value = (Camera_Mask * 16) + Light_ID`

#### A. 光源定義 (Light ID)

|**數值**|**燈光類型**|**說明**|
|---|---|---|
|**1**|**RE (Infrared)**|紅外光|
|**2**|**R (Red)**|紅光|
|**4**|**B (Blue)**|藍光|
|**3**|**G (Green)**|綠光|

#### B. 相機遮罩定義 (Camera Mask)

|**數值**|**對應層級**|**觸發相機**|
|---|---|---|
|**3** (0b0011)|**Layer 1**|同時觸發 Cam 1 & 2|
|**12** (0b1100)|**Layer 2**|同時觸發 Cam 3 & 4|

### 4. 系統操作邏輯 (Workflow)

為了配合 PLC 階梯圖中的步進接點（`LD= D113 K2`），系統採以下握手機制：

1. **初始同步**：CM4 啟動時會讀取 PLC 的 `M1000` 當前值並同步，不觸發拍照。
2. **觸發任務**：當 PLC 寫入 **40001** 且數值與上次不同時，CM4 立即將 **40003 改為 1 (Running)**。
3. **執行拍照**：Master 同時對對應的 Slave CM4 發送 HTTP 請求並存檔。
4. **回報完成**：任務結束後，CM4 將 **40003 改為 2 (OK)**，釋放 PLC 進入下一個步進狀態。
