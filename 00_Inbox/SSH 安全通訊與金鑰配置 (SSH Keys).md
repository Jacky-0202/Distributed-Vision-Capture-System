## 摘要

SSH 是一種加密網路協定，讓你的電腦與 GitHub 建立「免密碼」的信任關係。透過成對的公鑰與私鑰，確保資料傳輸的安全性。

---
## 核心組件：公鑰與私鑰

- **私鑰 (id_rsa)**：存放在你的電腦，絕對不能給別人。就像你的「印章」。
- **公鑰 (id_rsa.pub)**：上傳到 GitHub 設定中。就像是在 GitHub 留下的「印鑑證明」。
- **原理**：當你要上傳程式碼時，GitHub 會用公鑰出題，
	只有擁有私鑰的你才能解題，從而確認身分。

---
## 操作流程

1. **產生金鑰**：執行 `ssh-keygen -t ed25519 -C "your_email@example.com"`。
2. **取得公鑰**：打開 `id_rsa.pub`（通常在 `~/.ssh/` 資料夾），複製裡面的長串文字。
3. **部署 GitHub**：在 GitHub 設定頁面的 `SSH and GPG keys` 中點擊 `New SSH key` 並貼上。
4. **測試連線**：執行 `ssh -T git@github.com`。

---
## 核心價值

- **安全性**：比帳號密碼更難被駭客破解。
- **便利性**：設定一次後，之後的 `push` 與 `pull` 都不再需要重複輸入密碼。
