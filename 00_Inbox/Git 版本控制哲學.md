## 摘要

Git 就像是時光機與平行時空，它記錄的不是檔案，而是被紀錄的過程或是歷史紀錄，
它也像平行時空，同時擁有不同的版本分支來實驗新功能

---
## 核心概念：快照 (Snapshot) vs. 差異 (Delta)

- **傳統版本控制 (如 SVN)**：記錄檔案 A 在版本 2 改了哪幾行、檔案 B 在版本 3 刪了什麼。像是一張「修改清單」。
    
- **Git 的做法**：每次提交（Commit）時，Git 會對當下所有檔案「拍一張照片」。
    
    - 如果檔案沒變，Git 只會連結到之前的同一張照片，不重複佔空間。
    - 這讓 Git 在切換版本時極快，因為它是直接換成另一組照片，而不是回溯修改清單。

## 分散式架構 (Distributed)

- 每個人的電腦都有一個「完整的倉庫（Repository）」，包含所有歷史紀錄。
- **哲學意義**：不需要連網也能跳轉版本、開分支；伺服器壞了，任何一個人的電腦都能還原整個專案。

## 資料完整性 (Integrity)

- Git 使用 **SHA-1 雜湊演算法** 為所有內容計算識別碼（如：`4a5b6c...`）。
- **哲學意義**：
	檔案內容只要改一個字，識別碼就會完全不同。
	你無法在 Git 不知道的情況下修改歷史，這保證了紀錄的真實性。
