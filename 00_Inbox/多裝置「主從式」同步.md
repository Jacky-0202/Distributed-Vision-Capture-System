## 摘要

以電腦 A 為準的合併策略：
當電腦 A (主要) 與 電腦 B (次要) 的內容發生衝突時，確保電腦 A 的內容不被覆蓋，同時接收電腦 B 的新檔案。

## 核心邏輯

- **接收新檔案：** ✅ OK
- **內容衝突時：** 🛡️ 以電腦 A (本機) 為準

## 操作指令 (於電腦 A 執行)

#### 第一步：提交本地修改
```bash
git add .
git commit -m "Local update on primary device"
```


#### 第二步：強勢合併遠端內容

使用 `-X ours` 參數自動解決衝突：
```bash
git fetch origin
git merge -X ours origin/main
```

#### 第三步：更新 GitHub 紀錄

```bash
git push origin main
```
